Generated by `scripts/zip-etl/build-zip-index.ts` from the 2020 Census ZCTA
Gazetteer (fetched + sha256-verified by `fetch-zcta-gazetteer.sh`), with the
state precomputed by point-in-polygon against `data/us-state-polygons.geojson`
— the SAME polygon artifact the server clip seeds from (locked decision #6,
plan `2026-05-28-state-scope-selector`).

The "centroid misses" are ZCTAs whose population-weighted centroid (`INTPTLAT`/
`INTPTLONG`) falls just outside the 5%-simplified state polygon — typically
barrier-island, offshore, or coastal-sliver ZCTAs (e.g. `06390` Fishers
Island, NY). Because the clip and this precompute share one polygon artifact,
a dropped ZIP here is exactly one that the server clip would also resolve
empty — so dropping it (rather than snapping it to a nearest state) is the
correct, consistent behavior, and D5 surfaces it as "ZIP not recognized".

## ZIP ≠ ZCTA caveat (load-bearing)

This index is keyed by **ZCTA5** (ZIP Code Tabulation Area), not USPS ZIP. The
USPS issues ~41,000 ZIP codes; the Census publishes ~33,000 ZCTAs. The gap is
PO-box-only, military (APO/FPO), and single-point/large-customer ZIPs that have
no tabulation area. **A minority of perfectly valid USPS ZIPs therefore MISS
this index** (no entry → `lookupZip` returns `null`). That is expected and
handled explicitly downstream: D5's `ZipInput` shows a visible "ZIP not
recognized — try a nearby ZIP or pick a state" status rather than a silent
no-op. Do not "fix" misses by widening the source — the state selector is the
fallback by design.

## Encoding

Columnar, not array-of-objects:

```json
{ "v": 1, "states": ["US-AL", ...], "zips": { "01001": [42.06237, -72.62575, 1], ... } }
```

- `states[]` — deduped palette of `US-XX` codes; `zips[zip][2]` indexes into it
  (saves repeating the 5-char code on every one of 32,551 rows).
- `zips[zip]` — `[lat, lng, stateIdx]`, coordinates rounded to **5 decimals**
  (~1.1 m), matching the polygon coordinate precision so the two artifacts stay
  numerically aligned.
- `zips` keys are emitted in ascending ZIP order for a reproducible artifact.

## Regenerating

```sh
# 1. Fetch + verify the source (only network step; CI never runs this):
bash scripts/zip-etl/fetch-zcta-gazetteer.sh

# 2. Build the index (offline; reads .cache/ + data/us-state-polygons.geojson):
npx tsx scripts/zip-etl/build-zip-index.ts

# 3. Sanity-check the encoding on the 10-row fixture (no network):
npx vitest run scripts/zip-etl/build-zip-index.test.ts
```

If `data/us-state-polygons.geojson` is regenerated (Stream A / `#728`), rerun
step 2 so the precomputed states track the new clip exactly.
