# Design Agent 3: Accessibility-Shaped Designer

## Thesis

bird-maps.com already carries an exceptional, axe-validated accessibility skeleton — landmark order, native `<dialog>`, full WAI-ARIA tablist, focus-visible, inline-measured contrast — but its visual layer treats those structures as obligations rather than as the source of its aesthetic. The redesign's highest-leverage move is to **let the a11y substrate set the visual grammar**: every surface decision (palette, motion, focus ring, badge, modal) should be pulled directly from a decision the a11y story has already made. When the screen-reader narration and the visual hierarchy describe the *same* journey through the same words at the same beats, the result is both more accessible and more polished — the kind of polish that comes from a single intelligent system rather than from added flourishes. The goal is not "make Sky Atlas accessible." It is to draw Sky Atlas *out of* what's already accessibility-true. Below: 7 ideas where the a11y win is also the design win.

## Ideas

### Idea 1: Cluster-bubble palette as a measured-contrast triptych ("Sky / Sand / Ember")

The current `#51bbd6 / #f1f075 / #f28cb1` cluster colors clear AA against `#1a1a1a` text by accident — they were chosen visually, never arithmetically (canvas is excluded from axe). Sky Atlas should redesign the triptych with **measured ratios baked into the swatch names**: `Sky #6ec5d9 (8.2:1)`, `Sand #e8c060 (10.4:1)`, `Ember #e87a4a (5.1:1)` against `#1a1a1a` in light, mirrored against `#0d1424` in dark. Inline contrast comments — already a codebase convention at `styles.css:243-264` — extend into `observation-layers.ts` next to the paint expression. The palette reads as a deliberate atmospheric triptych (cool dawn → warm midday → ember dusk) that doubles as the cluster-density encoding. Dawn/dusk metaphor + cluster magnitude = same gesture.
- **The a11y win:** Cluster text contrast is now arithmetic, not vibes; CI doc convention extends to canvas paint; canvas-excluded axe gap is closed by a process commitment instead of a tool.
- **The design win:** Three named atmospheric colors instead of three arbitrary hex values; the cluster bubble becomes a *story* (small/medium/large = sky/sand/ember), not a heatmap. Names appear in tooltip + SR text identically.
- **Cost / risk:** Requires ~6 lines of paint expression edits and a one-page palette doc. If the family palette is also re-tuned (Idea 7), check both palettes together against the basemap mid-grey.

### Idea 2: "Two voices, one composition" — SR narration as sibling design artifact

Today, a sighted user sees a hierarchy (hero photo → name → meta → description → phenology); an SR user hears `aria-busy` on `<main>` plus `aria-live` regions in scattered components. The two stories don't correspond. **Redesign the surface schema so the visual hero stack and the SR reading order are byte-for-byte the same beats**: every visual H1 has a matching `aria-labelledby`; every visual sub-meta line has a `role="status"` peer when its content is data-derived; "filter applied: 14 days, notable, 47 results" is rendered as a single visible meta-line *and* as the sole `aria-live="polite"` text — the same string. The badge (Idea 4) renders the count visibly; the same string lives in `aria-live`. Authoring discipline: the surface designer writes one line of meta text, and the `role="status"` consumes it.
- **The a11y win:** Filter changes (currently silent — area-5 Finding 7) get a real announcement, view changes get a "Now showing Map of Arizona" hand-off. SR users get a coherent narrative instead of a stream of `aria-selected` flips.
- **The design win:** Forces every surface to have a *single* canonical meta-line — eliminates the 14 ad-hoc copy+class pairs (Finding 4.4) by collapsing them into one `<StatusLine>` primitive. Drives visual consistency the way a shared script drives film consistency.
- **Cost / risk:** Requires the `<StatusLine>` primitive (~80 lines) and a strict copy contract per surface. Discipline-load — easy to break when surfaces drift.

### Idea 3: Three modes, not two — "Day / Night / Stillness"

Sky Atlas proposes Day + Night. **Add a third equal-citizen mode: Stillness, the reduced-motion variant**. In Stillness, the dawn/dusk gradient hero on the detail surface freezes to a single representative tone (the basemap-tile mid-tone matched to the hour), MapLibre `easeTo` becomes `jumpTo`, the skeleton-shimmer becomes a single-step opacity tween (or a static border pulse), and the filter-sheet slide-up becomes an instant cross-fade. Crucially, Stillness is **selectable by the user** in the chrome — three icons: sun / moon / hourglass — *not* solely OS-derived. OS preference auto-selects on first visit; user override persists in localStorage.
- **The a11y win:** Reduced-motion isn't an afterthought-CSS-rule — it's a first-class branded mode with a label, an icon, and visible status. MapLibre `easeTo` reduced-motion leak (G5) closes deliberately. User control over motion = WCAG 2.3.3 + 2.2.2 compliance with breathing room.
- **The design win:** Three modes is more interesting than two — the brand has a contemplative-still register, useful for users in low-bandwidth-attention contexts (commute, classroom, with-screen-reader) regardless of vestibular need. "Stillness" is a *concept word* in Sky Atlas's voice register; it earns its place.
- **Cost / risk:** Adds a third theme branch (CSS custom property scope) and a third tested viewport state (axe coverage matrix grows from 13 to ~17). Worth it.

### Idea 4: Filter-active badge as the only number on the chrome

Pattern A (filter-sheet trigger) needs a filter-active count badge or it silently regresses (Risk R3). Make this badge **the only numeric character on the chrome at all** — no nav counts, no notification dots, no inbox badges. When zero filters are active, no badge renders; when 1+ are active, a single small filled circle with the count appears. Same pixel: visible filter-state and SR-announced filter-state are the same string. Badge color = `accent-primary` in both modes; size = 16px circle (above 14px minimum; below tap-target-size deliberately, since the badge is a status indicator and the parent button is the tap target). The number IS the only chromatic emphasis on the chrome.
- **The a11y win:** Closes the silent global-filter coupling problem; converts an invisible system into a discoverable one. Number is real text inside a `<span aria-label="3 filters active">`, not an SVG/canvas — readable.
- **The design win:** The number becomes a *graphic* — the chrome's only highlight. Visual restraint elsewhere (no other numbers, no other accent-color uses) makes the badge function as a deliberate accent, not as noise. Restraint = polish.
- **Cost / risk:** Requires Recommendation 5 spec ("what counts as active"). Trivial pixel cost.

### Idea 5: Focus ring as an inverse-color aperture, not an outline

Today's focus-visible is a uniform 2px `--color-text-strong` outline — workmanlike, defensible, but visually equivalent to "the operating system's focus ring." Sky Atlas's focus-visible should be **a 2px ring whose color is computed as the *inverse-luminance* of the immediate background** (`text-strong` on light surfaces, `text-strong` on dark surfaces, but with an additional `outline-offset` that creates a 2px `bg-page`-colored gap between the element and the ring — a "halo"). The halo reads as a deliberate aperture; the ring as a sharp seal. On colored surfaces (the Notable chip, the active tab, a primary button), the ring color flips to ensure 3:1 contrast against the surface — implemented via CSS `color-mix(in srgb, var(--text-strong) 100%, transparent)` keyed off a per-element `--focus-on-bg` token.
- **The a11y win:** 3:1 focus-ring contrast (WCAG 2.4.11/2.4.13 — Focus Appearance, new in 2.2) is mathematical, not eyeballed. Works against every surface including the new orange `accent-primary` (chip-active state) and dark mode.
- **The design win:** Focus becomes *part of the brand* — a small, distinctive, atmospheric aperture that reads as "this is the bird-maps.com keyboard story" the moment you land on a surface. Most websites' focus rings are an apology; Sky Atlas's is a flourish.
- **Cost / risk:** Adds one CSS custom property and one `color-mix` rule per interactive element. Browser support: `color-mix` is Safari 16.2+, Chrome 111+, Firefox 113+ — universal as of mid-2024.

### Idea 6: Map keyboard reachability via a "cluster manifest" sidebar

Today, clusters at zoom <14 are mouse-only — a keyboard user cannot tab into a 750-bird cluster. Spider-leaves only become tabbable after zoom-in. **Render an off-screen-but-tab-reachable "cluster manifest" list** on the map view: a `<section aria-label="Clusters in current view">` that lists each visible cluster (`<button>Sky Pinaleño Mountains, 247 birds</button>`) in geographic-name order. Activating an entry triggers the same `flyTo + cluster expand` action a click does. The list is visually rendered as a **collapsed left-rail tray** (not the FamilyLegend overlap territory — that's bottom-left); 36px wide collapsed showing tiny cluster-density bars; expanded on hover OR keyboard focus to reveal the full named list. The list is ALSO the primary mobile-keyboard surface, where the cluster manifest is what the skip-link skips TO.
- **The a11y win:** Keyboard parity with mouse on the map view's primary action (click cluster → zoom to it). Currently a hard gap (Finding 10). Skip-link target on map view becomes meaningful, not just compensatory.
- **The design win:** A geographic place-name list — "Pinaleño Mountains, Cave Creek Canyon, Patagonia Lake" — is a brand asset on its own. It teaches new users the named geography of Arizona birding, which the site implicitly knows but never tells. Doubles as discovery for people who don't know to zoom.
- **Cost / risk:** Requires reading visible-cluster state out of MapLibre (medium effort); place-name reverse-geocoding pipeline (the ecoregion data already gives region names). Trade-off: adds chrome surface area. Mitigation: collapsed-by-default, expands only on hover/focus.

### Idea 7: Color-independent state encoding ("dot-glyph" for notable + family-color shape pairing)

The notable filter is encoded only by color (`!` is a literal char on a colored chip — `ObservationFeedRow.tsx:81`); the family colors carry encoding as their *only* signal in cluster mosaic markers. **Pair every color with a non-color signal**: notable rows always carry a small filled-circle glyph at row-start (independent of any background); each family silhouette gets a paired *shape outline modifier* (square halo for hawks, rounded for songbirds, hexagonal for waterfowl) so a deuteranope user can read "hawk vs. wren" from shape alone. The family-legend renders the shape-modifier next to each color — the legend stops being only a color key and becomes a shape-and-color key.
- **The a11y win:** WCAG 1.4.1 (Use of Color) holds without depending on luminance differences alone; passes a greyscale screenshot test. Color-blind users (8% of male users — measurable cohort) can read state from grey-to-black PDFs.
- **The design win:** Shape pairing produces a *more* memorable and graphic family-encoding system, not less. Audubon's silhouettes are iconic because the shape carries identity, not because the color does. Sky Atlas's silhouettes earn the same reach.
- **Cost / risk:** Requires shape-modifier additions to the silhouette SVG paths (~20 lines of family_silhouettes.svgData edits). FamilyLegend visual gets one extra column. Worth auditing whether the basemap mid-grey vs. shape-outline produces a contrast issue.

## One bold direction (optional)

**The site's master metaphor is the SR narrative.** Stop treating the screen-reader story as a parallel obligation and start treating it as the *script* the visual designer animates. Every surface design begins by writing the SR narration — eight beats: brand → location → filter state → result count → primary content head → primary content body → secondary action → footer attribution — and the visual layout is the staging of those beats in space. If a visual element doesn't have a beat in the narration, it doesn't appear; if a narration beat doesn't have a visual, it gets one. The result is a site where the designer can listen to a VoiceOver pass and recognize their own composition. This is what "polished" means in 2026: the form is so disciplined that it survives modality-shift.
