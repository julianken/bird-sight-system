Notes & open questions for this section:
  • Type stack: currently shows the system-UI stack (kept). Sky Atlas's polish would benefit from a webfont (Inter, Söhne, GT America). Phase 1 noted zero web fonts as a perf baseline — adding one is a deliberate trade. Do you want a webfont?
  • Cluster bubble palette: the existing map clusters (#51bbd6 / #f1f075 / #f28cb1) are pink/yellow/cyan. Sky Atlas would want them warm-orange / warm-gold / cyan to match accents. They'd need re-contrast-checked against #1a1a1a count text in light, against #f5f7fb in dark.
  • Family-color palette: the 7 earth-tone family colors (DB-sourced from tokens.ts:124–158) need to coexist with this palette in both modes. Two compatible options: keep the earth tones (most parsimonious, but may feel under-saturated next to the orange accent in light), or commission a Sky-Atlas-tuned family palette (more cohesive, but requires DB migration). Either way the silhouettes need to read well on white AND deep navy basemaps.
  • Photo placeholder: the gradient is a stand-in. In real implementation, the iNat photo loads with aspect-ratio: 4/3 CSS already in place — no CLS regression.
